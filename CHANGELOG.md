# 更新日志 (Changelog)

Agent Status Board —— macOS 桌面悬浮组件 + 菜单栏指示灯，实时显示 Codex / Claude Code 的会话状态与用量。

版本号即 `VERSION` 文件，App 会按它做在线自更新（详见 README / AI-SETUP.md）。

---

## v1.22
- **新增:自动隐藏定时任务(自动化 / cron 触发的 Codex 会话)**。像"每日同步""每日备份"这类由定时器(launchd / cron)在固定时间自动跑的 Codex 任务,会挤占看板的"最近完成"、也不是你要盯的活。现在看板**自动识别并隐藏**它们:Codex 在 rollout 的 `session_meta` 里用 `thread_source: "automation"` 标记非交互(`codex exec` / 自动化)启动的会话,看板据此过滤,只保留你真正在终端里交互跑的会话。无需任何配置。

## v1.21
- **修复:Codex 额度数字偶尔偏低 / 和 Codex 实际额度对不上**。看板优先用 Codex 的实时用量接口(wham/usage),但之前**只要某一轮实时抓取抖动一下,就立刻退回本地 rollout 快照**——而快照的数字和实时限流值不是一回事(往往偏低,例如 5h 显示已用 81% 而实时其实 94%),于是看板会闪成那个偏低的错值。现在改为:**优先实时值;偶发失败时保留上一次的实时值,不再退回旧快照**(只有从未成功取过实时值时才用快照兜底)。另外补了用量抓取的日志(实时 / 兜底 / HTTP 状态),方便日后排查。

## v1.20
- **修复:会话名仍显示成自动生成的长标题**(例如你叫它"会议纪要",看板却显示"建立知识库会议纪要管理系统";"代码管理"显示成"列出文件夹下的代码仓库")。根因:Claude Code 对**没手动改过名**的会话**不存任何短标题**,只有冗长的自动 ai-title 和文件夹名——你心里的短名其实哪儿都没存。上一版从"显示文件夹名"改成"显示 ai-title",只是把一个不理想的名换成另一个。
  - 现在 `~/.agent-status-board/names.json` 除了**按路径**命名,还支持**按会话 ID 精确命名**(键写会话 UUID),且**优先级最高**。一个文件夹下有多个会话时(如 ~/LinkView 同时有 4 个),只有按会话 ID 才能分别命名——按路径会把同目录的会话全改成同一个名。
  - 解析优先级:会话 ID 置顶 → 路径置顶 → CC 自带标题(custom-title / ai-title) → 文件夹名。
  - 已为这两个会话写好置顶名;以后想给某个会话固定显示名,在 names.json 里加 `"<会话UUID>": "<显示名>"` 即可(也可继续在 Claude Code 里手动改名,看板会读到)。
- **修复:goal / 自动模式的会话「干完了还一直显示正在执行」**。goal 模式会自主连续干活,常常**不再触发 Claude Code 的 hook**,导致看板的事件记录冻结在"running"(可能还是一天前的状态)。现在对正在跑的 CC 会话**以 transcript 为准**判断:如果最后一轮是"已答完、没有待处理的工具调用"的助手回合、且静置超过 60 秒,就判定为已完成(收进"最近完成"),而不是一直黄着。同时,会话的保留/老化也改用 transcript 的真实活动时间而非 hook 记录时间,避免 hook 冻结把**还在干活的 goal 会话**误判为过期而删掉。

## v1.19
- **修复:会话标题显示成文件夹名**(例如"会议纪要"会话显示成"Obsidian")。CC 的会话标题(自动生成的 ai-title / 你设的 custom-title)常在 hook 触发之后才生成,导致看板停留在文件夹名。现在直接从 transcript 实时读取标题。
- **修复:正在跑的会话有时不显示**。上一版用 hook 时间戳判断"是否在跑",但 hook 安静一会儿(长任务 / MCP)就会被误判为已死而隐藏。现在改用 transcript 里"真实对话轮次"的时间(忽略标题生成等后台写入):长任务不再被误隐藏,只有真正结束 / 删除(无近期真实活动)的才隐藏。
- **改进:等待授权时转红**。CC 对 MCP 工具的授权请求常不触发 Notification 钩子,导致一直显示黄色。现在当一个工具调用挂起超过 90 秒还没返回结果时,判定为"等待你处理"并转红。

## v1.18
- **修复:已删除 / 已结束的会话还显示「正在执行」**。之前用 transcript 文件的修改时间判断会话是否在运行,但后台写入(标题自动生成、删除时碰到文件)会刷新这个时间,让已死 / 已删的会话看起来还在跑。改为只认 hook 自己的时间戳;hook 静默超过 15 分钟的「运行中」会话直接从看板移除(不再显示正在执行,也不会伪装成已完成)。
- **修复:点更新、下载完成后没有自动重启**。自更新脚本以 App 的 launchd 作业子进程身份运行,旧的重启方式(`launchctl unload`/`load`)会把作业重载进错误的域,导致 App 重装后不出现。改为显式定位用户 GUI 会话(`launchctl kickstart`/`bootstrap gui/<uid>`),并以 `open` 兜底,确保下载完成后稳定重启。

## v1.17
- **菜单栏角标只显示需要关心的**:右上角的任务数角标只统计「进行中 / 需处理」的任务,**已完成的不再计入**;某个工具全部完成时**不显示角标**,只留余量水球。小小的菜单栏不浪费在已经结束的事情上。

## v1.16
- **用量改为显示「余量(剩余)」**:菜单栏水球与桌面进度条都直接显示「还剩多少」(像电量 / 跟 Codex 一致)——CC 用了 17% 就显示「剩 83%」,不用自己算;满格绿 → 随消耗 → 黄 → 红(快耗尽)。
- **菜单栏水球美化**:竖向渐变绿(上浅下深)、放大到填满菜单栏高度、剩余数字加大加粗;进行中的任务数从「球旁孤立的数字」改成「贴在球右上角的状态色徽章」(绿/黄/红)。

## v1.15
- **菜单栏角标重做成「液态玻璃球」**：每个工具（Codex/CC）一个球；液面 = 该工具 5 小时用量（随用量往上涨，绿→黄→红）、球内数字 = 用量 %、球边数字 = 进行中任务数（按最高优先级状态着色）；亮色系、带水波晃动动画；放大到菜单栏高度。
- **菜单栏下拉重建**：点击角标弹出的下拉，改为与桌面悬浮组件一致的内容（需处理 / 正在执行 / 最近完成 / 用量），移除过时的「活跃会话 / 活动」布局。
- **清理"僵尸"会话**：被关闭/杀掉但没触发 Stop 的会话会一直卡在"正在执行"。现在超过 15 分钟没有真实活动（transcript/rollout 未更新）即判定为已结束，自动移出"正在执行"。
- **Codex 钩子不再每次更新都要重新信任**：根因是安装脚本每次都重写 config.toml 的 hooks 块，把 Codex 存在其中的 `[hooks.state]` 信任表一起删了。现在脚本对已就绪的钩子完全不改动（保留信任），必须改写时也会保留并外移该信任表；自更新会刷新钩子脚本但绝不碰 config.toml。
- **跨天保留最近完成**：底部"今日完成"改为"最近完成"，跨重启/跨天保留最近 5 个已完成会话（按会话去重、可点击回到会话继续、带"今天/昨天/日期"时间），明早一眼就能看到昨天干到哪、点一下接着干。
